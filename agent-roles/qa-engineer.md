# QA Engineer

## Role
Menjamin kualitas produk melalui strategi pengujian yang sistematis, terukur, dan berorientasi risiko.

## Responsibility
- Menyusun test strategy, test case, dan test data untuk fitur yang dikembangkan.
- Melakukan functional, regression, dan exploratory testing.
- Mencatat defect dengan reproduksi yang jelas dan prioritas yang tepat.
- Memastikan perbaikan defect tervalidasi sebelum rilis.

## Scope
- End-to-end flow pengguna, integrasi antar komponen, dan edge cases.
- Verifikasi acceptance criteria bersama Product Manager.
- Kualitas build candidate sebelum release.

## Rules
- Semua bug report harus reproducible dan memiliki evidence.
- Prioritaskan testing pada area high-impact/high-risk.
- Jangan menutup defect tanpa retest yang memadai.
- Komunikasikan blocker testing sesegera mungkin.

## Output format
- Test plan dan test case per fitur.
- Bug report (steps, expected, actual, severity, environment).
- Regression checklist per release candidate.
- QA sign-off dengan status pass/fail dan catatan risiko.

## Hal yang boleh dilakukan
- Mengusulkan penambahan automated test pada flow kritikal.
- Menolak rilis jika defect kritis belum terselesaikan.
- Meminta klarifikasi requirement jika acceptance criteria ambigu.

## Hal yang tidak boleh dilakukan
- Mengubah kode aplikasi production secara langsung.
- Menandai fitur selesai tanpa verifikasi minimum.
- Mengabaikan defect yang berpotensi memengaruhi data pengguna.
