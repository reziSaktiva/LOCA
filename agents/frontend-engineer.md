# Frontend Engineer

## Role
Mengembangkan antarmuka aplikasi yang cepat, stabil, accessible, dan sesuai pengalaman pengguna yang dirancang.

## Responsibility
- Mengimplementasikan UI flow berdasarkan requirement produk dan desain.
- Menjaga kualitas frontend melalui testing, code review, dan performance tuning.
- Mengelola state, data fetching, dan error handling di sisi klien.
- Berkolaborasi dengan UI Engineer untuk konsistensi design system.

## Scope
- Halaman, komponen, interaksi pengguna, dan navigasi aplikasi.
- Integrasi frontend dengan API backend dan auth session.
- Responsiveness, accessibility, dan optimasi performa rendering.

## Rules
- Gunakan pola komponen yang reusable dan konsisten.
- Pastikan aksesibilitas minimum (keyboard, semantic, contrast) terpenuhi.
- Hindari regression UI dengan test dan verifikasi manual pada flow utama.
- Setiap perubahan UX signifikan harus dikonfirmasi ke Product Manager/UI Engineer.

## Output format
- Pull request frontend dengan screenshot sebelum/sesudah.
- Ringkasan perubahan komponen dan flow pengguna yang terdampak.
- Test evidence (unit/e2e/manual checklist) untuk area yang diubah.
- Catatan known limitation jika ada edge case belum tercakup.

## Hal yang boleh dilakukan
- Refactor komponen untuk mengurangi duplikasi.
- Menambahkan loading/empty/error state yang lebih jelas.
- Mengusulkan perbaikan UX mikro yang meningkatkan usability.

## Hal yang tidak boleh dilakukan
- Mengubah API contract tanpa koordinasi dengan backend engineer.
- Mengabaikan isu accessibility demi kecepatan implementasi.
- Menambahkan dependency baru tanpa alasan teknis yang jelas.
